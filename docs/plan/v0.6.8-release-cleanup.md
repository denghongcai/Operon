# v0.6.8 Release Cleanup

Status: Completed.

This phase closes the final review findings before tagging v0.6.8.

## Scope

- Bound daemon runtime-only job maps so completed jobs do not grow memory
  without limit.
- Preserve audit context across job async task boundaries.
- Bring CI and README validation lists up to the current v0.6.8 smoke
  requirements.
- Replace stale daemon flag documentation with unified `config.yaml` guidance.
- Align the public protocol version and release examples with v0.6.8.

## Completed

- Completed job event broadcasters are removed after the terminal event is
  sent.
- Completed job log buffers are retained only up to a bounded in-memory limit;
  newer completed jobs remain queryable while older buffers can be evicted.
- Job async tasks are spawned inside the captured request audit context.
- CI now runs [`scripts/verify-v0.6.7-runtime.sh`](../../scripts/verify-v0.6.7-runtime.sh) as the v0.6.7/v0.6.8 runtime
  and protocol validation.
- [`scripts/verify-v0.6.7-runtime.sh`](../../scripts/verify-v0.6.7-runtime.sh) now covers streaming fs write, job stdin,
  binary log streaming, and list API callers.
- Runtime API docs and PROTOCOL.md now describe auth/listen configuration
  through unified `config.yaml`.
- `PROTOCOL_VERSION` and README release examples are aligned to v0.6.8.
