# v0.2 Acceptance

v0.2 hardens the MVP without changing the core boundary: Operon still runs on top of existing private connectivity and does not implement a VPN, relay, NAT traversal, or mesh IP layer.

Historical note: this file describes the v0.2 implementation at the time it was
accepted, including temporary HTTP endpoints and daemon flags. The current
runtime uses unified `config.yaml`, gRPC, SDK/generated clients, and the
protocol documented in `PROTOCOL.md`.

## Acceptance Scope

- Filesystem transfer can avoid JSON string payloads with raw-body read/write endpoints and CLI `--output` / `--file`.
- Job output can be followed through CLI polling with `job logs --follow`.
- Daemon endpoints can require bearer token authentication with `--auth-token` or `--auth-token-file`.
- CLI node config can carry per-node bearer tokens.
- TypeScript SDK endpoints can carry per-node bearer tokens.
- Daemon errors return structured `{ code, message }` JSON bodies.
- Network provider support is explicit endpoint resolution, not connectivity.
- Daemon can append audit and final job records to a local JSONL store with `--store`.
- Graph execution can write a trace file with `--trace-output`, and `operon trace show` can print it.
- Jobs can request scoped secrets by name, and policy decides which names may be injected into the job environment.
- TypeScript SDK graph steps can request scoped secrets.
- Audit records secret names only, not secret values.

## Canonical Validation

```bash
pnpm install --frozen-lockfile
cargo fmt --check
cargo check --workspace --locked
cargo test --workspace --locked
cargo clippy --workspace --locked -- -D warnings
pnpm typecheck
pnpm -r test
scripts/verify-v0.2-docker.sh
```

## Known Limits

- The raw fs endpoints currently avoid JSON encoding but still buffer a complete request or response body in the daemon.
- Job stdin is not yet a streaming API. v0.2 supports streaming-like stdout/stderr consumption through followed logs.
- The CLI currently speaks HTTP only. HTTPS and gRPC clients remain post-v0.2 protocol decisions.
- Provider adapters do not discover nodes yet; they resolve manually configured endpoints.
- The JSONL store is append-only and is not yet a queryable database.
