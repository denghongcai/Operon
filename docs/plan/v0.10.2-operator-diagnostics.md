# v0.10.2 Operator Diagnostics

Status: Completed.

## Purpose

Provide one operator-facing diagnostic entrypoint for common config, endpoint,
auth, protocol, policy, and service readiness problems.

## Scope

- Add `operon doctor`.
- Report unknown `config.yaml` fields without failing the command when the
  remaining config is valid.
- Resolve configured endpoints and surface token/file/env auth errors.
- Check runtime health and compare daemon `Health.version` with the CLI protocol
  version.
- Reuse daemon-owned `ExplainCapability` diagnostics instead of duplicating
  policy logic in the CLI.
- List configured services and run policy-aware service health checks when
  service check permission is granted.
- Support human output and `--json`.
- Add validation coverage in
  [`scripts/verify-v0.10.2-operator-diagnostics.sh`](../../scripts/verify-v0.10.2-operator-diagnostics.sh).

## Completed

- `crates/operon-cli/src/commands/doctor.rs` implements the diagnostic report.
- `operon doctor --help` is exposed as a top-level CLI command.
- JSON output includes config warnings, endpoint/auth errors, health/protocol
  status, capability diagnostics, and service checks.
- README, DEVELOPMENT, AGENTS.md, and repo-local skills point operators and
  agents at `operon doctor` for first-pass troubleshooting.
