# v0.18.11 Release Gate Orchestration Cleanup

Status: Completed.

## Goal

Make the release publication gate order executable and auditable from one
repo-local entrypoint so release work no longer depends on scattered manual
workflow knowledge.

## Scope

1. Add a release gate orchestration helper that prints the required pre-tag and
   post-publication commands for a public release tag and exact commit.
2. Add a pre-tag verification mode that checks successful CI, CodeQL,
   cross-platform live mount release gates, and Windows Runner Image Smoke on
   the exact release commit.
3. Add a post-release verification mode that checks the Draft Release workflow,
   public release state, release artifact verification, downloaded release
   install usability, and README Quickstart verification.
4. Keep existing workflow implementations and action versions unchanged; this
   phase only orchestrates and verifies the gates that already exist.
5. Update release CI observability docs, DEVELOPMENT release guidance,
   AGENTS.md, and focused validation coverage.

## Acceptance

- `scripts/release-gate-orchestrate.sh plan <tag> <commit> [repo]` prints the
  complete gate order with exact `gh` and `git` commands.
- `scripts/release-gate-orchestrate.sh pretag <tag> <commit> [repo]` verifies
  that CI, CodeQL, live mount release gates, and Windows Runner Image Smoke
  passed on the exact release commit.
- `scripts/release-gate-orchestrate.sh postrelease <tag> <commit> [repo]`
  verifies that the release is public and that post-publication verification
  workflows passed on the exact release commit.
- Focused validation is wired into the consolidated `core` validation group.
