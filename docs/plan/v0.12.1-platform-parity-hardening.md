# v0.12.1 Platform Parity Hardening

Status: Completed.

Goal: close the highest-risk macOS and Windows behavior gaps after the release
target decision, without pretending that mount adapters or Linux-only workspace
hardening have full cross-platform parity.

## Scope

This phase should harden platform-specific runtime behavior that matters for a
credible macOS and Windows core runtime preview.

Planned:

1. Windows private file semantics.
   - Audit token/config private-file checks on Windows.
   - Either implement Windows ACL-based privacy checks for generated secrets
     and token files, or surface an explicit warning that Unix mode checks do
     not apply.
   - Keep Unix private-file behavior unchanged.

2. Windows exec cancellation semantics.
   - Assess current Windows process cancellation behavior for non-interactive
     exec.
   - If cancellation cannot reliably terminate child process trees, add a
     Windows Job Object based implementation for parity with Unix process-group
     termination.
   - If full parity is deferred, document the narrower Windows guarantee in
     runtime docs and doctor output.

3. macOS PTY validation and explicit Windows PTY validation status.
   - Keep `portable-pty` as the only PTY abstraction.
   - Add short-lived interactive session smoke tests that validate session
     start, output, resize where supported, and exit on Unix-like CI runners.
   - Report Windows PTY validation as deferred if Windows CI cannot exercise it
     reliably without hanging.

4. Platform-aware diagnostics.
   - Extend `operon doctor` to report platform caveats where they affect user
     expectations:
     - Linux-only mount support.
     - Windows private-file ACL or warning state.
     - Windows exec cancellation guarantee.
     - macOS PTY validation and Windows PTY validation state.
     - Service forwarding firewall caveats.

5. Documentation and validation.
   - Update README, `PROTOCOL.md`, runtime API docs, and architecture docs for
     any platform guarantee that changes.
   - Add a validation script for platform parity hardening and wire it into CI.
   - Keep docs/help/skills sync validation current when CLI output or help
     changes.

## Non-Goals

- No macFUSE or WinFsp mount adapter implementation.
- No new PTY dependency beyond `portable-pty`.
- No broad rewrite of exec/session protocol envelopes.
- No new networking ownership or firewall automation.
- No claim that macOS or Windows workspace containment equals Linux
  `openat2(RESOLVE_BENEATH)`.

## Done When

- Windows private token/config handling has either ACL semantics or explicit
  diagnostics and docs.
- Windows exec cancellation has a documented and tested guarantee.
- macOS PTY session behavior is smoke-tested before any release claim depends
  on it, and Windows PTY validation remains explicitly deferred until it has a
  non-hanging runner-safe smoke path.
- `operon doctor` reports platform caveats that users need for first-pass
  troubleshooting.
- Platform parity docs, CLI help, and validation scripts agree.

## Completed

- Added platform-specific private-file diagnostics. Unix reports owner-only
  mode protection; Windows reports an ACL warning instead of claiming Unix mode
  parity.
- Documented Windows non-interactive exec cancellation as direct-child
  best-effort until Job Object based process-tree cancellation is implemented.
- Added a daemon platform guarantee test for exec cancellation semantics.
- Added `portable-pty` smoke validation for session start, resize, output, and
  exit behavior on Unix-like CI runners; Windows PTY validation is reported as
  deferred instead of blocking CI on a hanging smoke test.
- Extended `operon doctor` reports with platform caveats for mount support,
  private-file protection, exec cancellation, PTY validation, and
  firewall-sensitive service forwarding.
- Updated README, `PROTOCOL.md`, and runtime API docs with the platform
  guarantees and caveats.
- Added `scripts/verify-v0.12.1-platform-parity-hardening.sh` and wired it into
  CI and DEVELOPMENT.md.

## Remaining

- No v0.12.1 work remains.
- Windows Job Object process-tree cancellation remains future hardening.
- macFUSE and WinFsp mount adapters remain deferred.
