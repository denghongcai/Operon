# v0.15.1 Release Gate Hardening

Status: Completed.

## Goal

Move release archive failures into the draft release workflow so public
verification confirms an already-hardened artifact instead of being the first
place packaging regressions appear.

## Scope

- Smoke each generated Rust release archive before upload.
- Keep Linux cross-compiled arm archives structurally verified while
  smoke-running native executable archives.
- Verify macOS archives without relying on runner `DYLD_*` environment.
- Make the macOS FUSE-T packaging contract explicit: `libfuse-t.dylib` must be
  present and packaged `operon` must have an `@executable_path` rpath.
- Rename the release-only live mount gate to a cross-version workflow while
  keeping compatibility with older v0.14 gate evidence.

## Completed

- Added `scripts/smoke-release-archive.sh` for archive extraction, packaged
  file checks, macOS FUSE-T dylib/rpath checks, and packaged binary smoke.
- Updated `release-draft.yml` to run archive smoke before uploading Rust
  release archives. Native Linux x86_64, macOS, and Windows packages run
  binaries from the extracted archive; Linux arm64/armv7 cross-build packages
  run structural archive checks.
- Updated `scripts/verify-release-artifacts.sh` to reuse the same archive smoke
  script during public release artifact verification.
- Added `scripts/verify-release-gates.sh` as the generic release gate checker
  and kept `scripts/verify-v0.14-release-gates.sh` as a compatibility wrapper.
- Renamed the manual live mount workflow display name to
  `Cross-Platform Live Mount Smoke` while allowing gate verification to accept
  older `v0.14 Live Mount Smoke` runs for historical tags.
- Added validation coverage through
  `scripts/verify-v0.15.1-release-gate-hardening.sh`.

## Remaining

- No v0.15.1 release gate hardening work remains.
