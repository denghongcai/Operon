# v0.17.2 Mount Probe Reliability Cleanup

Status: Completed.

## Goal

Keep the cross-platform live mount workflow green for release-relevant mount
gates while preserving the fuser hello probe as an opt-in macOS FUSE-T
diagnostic.

## Scope

1. Treat `macos-fuser-hello` as a strict standalone diagnostic probe.
2. Treat the fuser hello probe as diagnostic-only when `platform=all` is used,
   so third-party fuser clean-unmount or short-read behavior does not fail the
   release-oriented aggregate workflow.
3. Keep macOS FUSE-T live mount, Windows WinFsp live mount, fuse-zip, and
   libfuse low-level probes strict in `platform=all`.
4. Add validation coverage for the workflow and script contract.

## Acceptance

- `platform=all` can finish green when only the fuser hello diagnostic exits
  non-zero.
- `platform=macos-fuser-hello` still returns the real fuser probe exit code.
- The phase plan and validation script document the distinction between release
  gates and diagnostic probes.

## Progress

- Added a diagnostic-only mode to the fuser hello probe workflow step when the
  aggregate `platform=all` live mount smoke is used. The step now records the
  real fuser exit code and emits a GitHub warning, but does not fail the
  aggregate release-oriented workflow for fuser-only diagnostic failures.
- Kept `platform=macos-fuser-hello` strict by returning the raw fuser probe
  exit code outside aggregate mode.
- Added `scripts/verify-v0.17.2-mount-probe-reliability.sh` and wired it into
  the consolidated `core` validation group.
