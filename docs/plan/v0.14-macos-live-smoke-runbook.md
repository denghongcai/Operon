# v0.14 macOS Live Mount Smoke Runbook

Status: blocked until a macOS host with a working macFUSE runtime is available.

## Purpose

This runbook is the release-gate procedure for the remaining v0.14 macOS live
mount validation. GitHub-hosted `macos-latest` remains useful for diagnostics,
but it has not provided a usable macFUSE FSKit or loaded kernel backend for the
full live mount smoke.

## Why Not GitHub-Hosted macOS

The hosted runner can install macFUSE and can reach Operon's mount start path,
but it has not satisfied the system-level macFUSE runtime requirements:

- FSKit backend: hosted-runner unified logs reported `Hello FSClient!
  entitlement no`, followed by macFUSE daemon mount and server-advertise
  failures.
- Kernel backend: hosted-runner logs reported `macFUSE kernel extension is not
  loaded`, and the live smoke never exposed the seed file.

Treat GitHub-hosted macOS as a diagnostic lane only. It is not the v0.14 macOS
release gate unless a future runner image proves it can provide a working FSKit
or loaded kernel backend.

## Required Host

Use a macOS host that satisfies one of these backend paths:

- FSKit backend: macOS 15.4 or newer, macFUSE installed, and
  `pkg-config fuse` available.
- Kernel backend: macFUSE installed, approved, and the macFUSE kernel extension
  loaded.

For GitHub Actions, register the host as a repository self-hosted runner with
all of these labels:

- `self-hosted`
- `macOS`
- `macfuse`

Do not dispatch the `self-hosted-macfuse` workflow lane until the runner is
online. If no matching runner exists, the run will queue instead of validating
the release gate.

## Host Preflight

Run the preflight directly on the host before dispatching the full smoke:

```bash
OPERON_MOUNT_MACOS_BACKEND=fskit scripts/preflight-v0.14-macos-macfuse-host.sh
```

For a kernel-backend host:

```bash
OPERON_MOUNT_MACOS_BACKEND=kernel scripts/preflight-v0.14-macos-macfuse-host.sh
```

The preflight must pass before the host can satisfy the v0.14 release gate. It
fails early for missing macFUSE, missing `pkg-config fuse`, unsupported FSKit
macOS versions, and unloaded kernel backends.

## Workflow Dispatch

After the runner is online and the preflight passes, dispatch the live mount
workflow from the current `main` commit:

```bash
gh workflow run "v0.14 Live Mount Smoke" \
  --repo denghongcai/Operon \
  --ref main \
  -f platform=macos \
  -f macos_runner=self-hosted-macfuse \
  -f macos_backend=fskit
```

Use `-f macos_backend=kernel` only on a host where the macFUSE kernel extension
is already approved and loaded.

## Success Evidence

The macOS release gate is satisfied only when the workflow completes
successfully on the self-hosted macOS/macFUSE lane and the smoke output reaches:

```text
v0.14 macOS live mount smoke passed
```

Record the workflow run ID in `docs/plan/development-phases.md` and
`docs/plan/v0.14-cross-platform-live-mount.md` before publishing the release.

## Failure Handling

If the smoke fails, do not wait for repeated retries without new evidence.
Open the failed run, fetch the macOS live mount artifact, and inspect:

- preflight output
- `macOS live mount smoke exit code`
- daemon log
- mount log
- recent FSKit/LiveFS/macFUSE unified logs

If a run is queued because no matching runner exists, cancel it and fix runner
registration before dispatching again.
