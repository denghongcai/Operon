# v0.6 Acceptance

v0.6 is scoped to real Linux mount support.

It includes:

- Linux-only FUSE mount adapter.
- mount behavior over the existing Operon fs protocol.
- path confinement to configured remote mounts.
- policy enforcement through the remote daemon.
- audit records for mounted fs operations.
- CLI UX for creating and cleaning up live mounts.

It deliberately does not include WinFsp, macOS mount support, offline sync,
distributed cache invalidation, multi-writer conflict resolution, or using FUSE
as a separate capability model.

## Validation

The canonical v0.6 validation should run:

```bash
cargo fmt --check
cargo check --workspace --locked
cargo test --workspace --locked
cargo clippy --workspace --locked -- -D warnings
scripts/verify-v0.6-linux-mount.sh
```

The Linux mount validation must prove:

- `operon mount <node:/path> --to <dir>` creates a live mount.
- listing mounted directories reflects remote node content.
- reading mounted files returns remote file content.
- path escapes outside the configured remote mount are denied.
- denied mounted operations are audited.
- unmount and signal cleanup are reliable.
- validation clearly reports missing host requirements such as `/dev/fuse`.

## Known Limits

- v0.6 targets Linux only.
- The first real mount may be read-only unless the mount contract explicitly
  accepts write behavior.
- Mount support remains an adapter over the fs protocol, not a replacement for
  the daemon fs capability.
