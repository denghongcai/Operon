# v0.6 Acceptance

v0.6 is scoped to real Linux read-only mount support.

It includes:

- Linux-only read-only FUSE mount adapter.
- mount behavior over the existing Operon fs protocol.
- path confinement to configured remote mounts.
- policy enforcement through the remote daemon.
- audit records for mounted fs operations.
- CLI UX for creating and cleaning up live mounts.

It deliberately does not include WinFsp, macOS mount support, offline sync,
write support, distributed cache invalidation, multi-writer conflict resolution,
or using FUSE as a separate capability model.

## Mount Contract

v0.6 mount behavior is intentionally narrow:

- `operon mount <node:/path> --to <dir>` creates a foreground live FUSE mount.
- The mount root maps exactly to the configured remote path.
- Remote paths must be absolute and cannot contain `..`.
- FUSE child lookup rejects empty names, `.`, `..`, slash-containing names, and
  non-UTF-8 names.
- Directory listing, metadata lookup, and file reads call the existing daemon fs
  gRPC API.
- File contents are read through the gRPC streaming `ReadFile` protocol and
  sliced to the kernel requested range.
- Metadata is cached by the kernel for one second.
- File permissions are exposed as read-only: directories `0555`, files `0444`.
- Write-like operations are unsupported in v0.6 and are handled by the read-only
  mount contract, not by a new capability model.
- Remote policy remains authoritative. If the daemon denies stat/list/read, the
  mount returns the mapped FUSE error and the daemon records the audit event.

## Validation

The canonical v0.6 validation should run:

```bash
cargo fmt --check
cargo check --workspace --locked
cargo test --workspace --locked
cargo clippy --workspace --locked -- -D warnings
scripts/verify-v0.6-linux-mount.sh
```

The Linux mount validation must prove:

- `operon mount <node:/path> --to <dir>` creates a live mount.
- listing mounted directories reflects remote node content.
- reading mounted files returns remote file content.
- path escapes outside the configured remote mount are denied before mounting.
- daemon-denied mounted operations are audited by the remote node.
- unmount and signal cleanup are reliable.
- validation clearly reports missing host requirements such as `/dev/fuse`.

## Known Limits

- v0.6 targets Linux only.
- v0.6 is read-only. Write mount behavior is deferred to v0.6.1.
- Mount support remains an adapter over the fs protocol, not a replacement for
  the daemon fs capability.
