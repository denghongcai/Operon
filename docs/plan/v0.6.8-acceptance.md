# v0.6.8 Acceptance Plan

Status: Completed.

v0.6.8 is a gRPC schema stabilization milestone before v0.7 TUI work. It turns
existing runtime behavior into stronger protobuf contracts and removes legacy
schema ambiguity.

## Scope

v0.6.8 includes:

- typed protobuf enums for runtime enum fields.
- proto3 optional presence instead of manual `has_*` flags.
- pruning or quarantining legacy non-compiled proto files.
- explicit streaming request envelopes for file write and job stdin.
- pagination fields for list APIs.

v0.6.8 does not include:

- TUI console implementation.
- new capabilities.
- policy/config package extraction.
- `openat2` workspace traversal.
- protocol-level `argv[]` job execution.
- endpoint discovery beyond local mDNS.

## Acceptance Criteria

Typed enums:

- `Capability.kind`, `JobRecord.status`, `JobEvent.status`, and service
  protocol use protobuf enum fields.
- Rust and TypeScript clients cannot accidentally send arbitrary strings for
  those fields through generated types.
- CLI and JSON output keep stable human-readable names.

Optional presence:

- optional timeout, exit code, reason, run id, and step id fields use proto3
  presence.
- removed `has_*` fields are reserved or otherwise protected from accidental
  reuse.
- omitted fields preserve current runtime behavior.

Proto surface:

- inactive proto files are either removed or clearly archived.
- build scripts compile only active runtime protocol files.

Streaming envelopes:

- `WriteFile` and `WriteJobStdin` use explicit metadata/data message variants.
- daemon validation still rejects missing metadata, duplicate metadata, and
  target switches.
- CLI and SDK streaming writes continue to work.

Pagination:

- list request messages include `page_size` and `page_token`.
- list responses expose `next_page_token` where pagination applies.
- CLI and high-level SDK helpers preserve current complete-list behavior by
  walking pages internally.

## Validation

Required validation commands:

```bash
cargo fmt --check
cargo clippy --workspace --locked -- -D warnings
cargo test --workspace --locked
pnpm --filter @operon/sdk typecheck
pnpm --filter @operon/sdk test
scripts/verify-v0.5-docker.sh
scripts/verify-v0.6.7-runtime.sh
```

Runtime smoke validation must cover:

- fs write streaming.
- job stdin streaming.
- job log streaming.
- list capabilities/jobs/services/audit through paginated APIs.

## Remaining After v0.6.8

- v0.7 CLI TUI console.
- policy/config package boundary cleanup.
- Linux `openat2` or fd-relative workspace traversal.
- optional protocol-level `argv[]` shell-free job execution.
