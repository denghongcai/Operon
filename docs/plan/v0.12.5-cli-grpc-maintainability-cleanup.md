# v0.12.5 CLI gRPC Maintainability Cleanup

Status: Completed.

## Goal

Continue behavior-preserving maintainability cleanup by splitting the remaining
large CLI gRPC helper surface into focused modules before more protocol and
operator features accumulate there.

## Context

v0.12.2 split daemon runtime routing and CLI exec/session responsibilities.
The next largest user-facing maintainability hotspot is
`crates/operon-cli/src/grpc.rs`, which still mixes connection setup, metadata,
stream handling, node/client helpers, and command-family request wrappers.

This phase should reduce the cost of future CLI protocol changes without
changing public CLI behavior.

## Scope

1. Responsibility map.
   - Reassess `crates/operon-cli/src/grpc.rs` and identify stable clusters such
     as channel construction, auth metadata, node selection, filesystem streams,
     exec streams, service tunnel helpers, and diagnostics helpers.
   - Keep extracted modules under CLI ownership unless a helper is already
     shared through `operon-grpc-client`.

2. Behavior-preserving module split.
   - Move focused helper clusters behind small CLI modules with narrow public
     functions.
   - Avoid changing command output, JSON contracts, stream semantics, or error
     wording unless tests prove the current behavior is wrong.
   - Preserve existing shared-client crate boundaries.

3. Tests and validation.
   - Move existing unit tests with the code they validate.
   - Add static validation that rejects re-growth of unrelated helper clusters
     in `grpc.rs`.
   - Run focused CLI contract and protocol validation scripts affected by the
     split.

4. Documentation.
   - Update `docs/plan/development-phases.md` with the final module boundaries.
   - Update AGENTS guidance if any command/help/docs synchronization rule
     changes.

## Non-Goals

- No runtime protocol changes.
- No CLI UX redesign.
- No config command rendering split unless it becomes necessary to complete the
  gRPC helper cleanup.
- No SDK API changes.

## Acceptance Criteria

- `crates/operon-cli/src/grpc.rs` is reduced to a narrow orchestration or
  compatibility surface.
- Extracted modules have clear responsibilities and colocated tests.
- Existing CLI JSON, quiet, stream, and exit-code contracts continue to pass.
- CI includes a maintainability validation script for the new CLI gRPC module
  boundaries.
- `docs/plan/development-phases.md` records what changed and what remains.

## Completed

- Kept `crates/operon-cli/src/grpc.rs` as the compatibility and shared
  connection/context surface.
- Moved filesystem helpers into `crates/operon-cli/src/grpc_fs.rs`.
- Moved non-session exec helper RPCs into
  `crates/operon-cli/src/grpc_exec_api.rs`.
- Moved service list/check helpers into
  `crates/operon-cli/src/grpc_service_api.rs`.
- Moved audit listing into `crates/operon-cli/src/grpc_audit.rs`.
- Added `scripts/verify-v0.12.5-cli-grpc-maintainability-cleanup.sh` and wired
  it into CI and DEVELOPMENT.md.

## Remaining

- No v0.12.5 work remains.
- Config command rendering and deeper onboard splitting remain future
  maintainability candidates.
