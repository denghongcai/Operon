# v0.14.1 Mount Stabilization

Status: Completed.

## Goal

Stabilize the cross-platform live mount surface after v0.14 by tightening
adapter error semantics and validation coverage without changing the mount
protocol or release workflow.

## Scope

1. POSIX-style adapter error mapping.
   - Classify remote `ALREADY_EXISTS` filesystem failures distinctly from
     unknown adapter failures.
   - Map that classification to `EEXIST` for Unix FUSE callbacks so create and
     mkdir collisions do not surface as generic I/O errors.

2. Validation.
   - Add focused unit coverage for the platform-neutral classification and Unix
     errno mapping.
   - Add a dedicated v0.14.1 validation script and wire it through the
     consolidated validation runner.

## Non-Goals

- No new mount protocol messages.
- No cache invalidation, offline sync, or conflict-resolution engine.
- No change to release gating; v0.14 live-smoke gates remain the release
  requirement for cross-platform mount behavior.

## Acceptance Criteria

- Remote `ALREADY_EXISTS` errors are classified as a stable mount error kind.
- Unix FUSE callbacks map that kind to `EEXIST`.
- `scripts/verify-v0.14.1-mount-stabilization.sh` passes and is wired into
  `scripts/ci/run-validations.sh`.

## Completed

- Added `MountErrorKind::AlreadyExists` and mapped
  `tonic::Code::AlreadyExists` to that mount classification.
- Updated Unix FUSE errno mapping to return `EEXIST` for existing-path create
  or mkdir collisions.
- Added focused tests for mount-core classification and FUSE errno conversion.
- Added consolidated validation coverage through
  `scripts/verify-v0.14.1-mount-stabilization.sh`.

## Remaining

- No v0.14.1 mount stabilization work remains.
