# v0.9.3 Store-Backed Audit Visibility

Status: Completed.

Goal: make daemon audit inspection restart-safe by loading persisted audit
events from the existing append-only JSONL store at startup.

## Scope

- Load `kind: audit` records from the configured store during daemon startup.
- Keep `audit list` and `audit show` backed by the same bounded in-memory queue
  after startup.
- Preserve the existing append-only store format and job reload behavior.
- Keep audit retention bounded by `MAX_IN_MEMORY_AUDIT_EVENTS`.

## Non-Goals

- No new query database or indexed store.
- No protocol changes.
- No change to audit event schema.
- No change to job-log persistence semantics.

## Completed

- Added `operon_store::load_audit_events`.
- Updated daemon startup to seed `AppState.audit` from persisted audit events.
- Added bounded startup retention through `bounded_audit_events`.
- Added focused tests for persisted audit loading and bounded queue retention.
- Added `scripts/verify-v0.9.3-store-backed-audit-visibility.sh`.
