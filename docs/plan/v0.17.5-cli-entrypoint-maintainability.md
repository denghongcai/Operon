# v0.17.5 CLI Entrypoint Maintainability Cleanup

Status: Completed.

## Goal

Reduce `operon-cli` entrypoint responsibility without changing public CLI
commands, flags, output, config loading, or command behavior.

## Scope

1. Move clap argument and subcommand definitions out of
   `crates/operon-cli/src/main.rs`.
2. Move top-level command dispatch and shell completion handling out of
   `main.rs`.
3. Keep public help, command names, aliases, flags, JSON/quiet behavior, and
   existing command modules unchanged.
4. Add focused validation for the new CLI entrypoint boundaries.

## Acceptance

- `crates/operon-cli/src/cli_args.rs` owns the public clap model.
- `crates/operon-cli/src/cli_dispatch.rs` owns top-level command dispatch and
  completion generation.
- `main.rs` remains a thin parse-and-dispatch entrypoint.
- Existing CLI integration tests and focused CLI maintainability validation
  pass.

## Progress

- Moved the public clap model into `crates/operon-cli/src/cli_args.rs`.
- Moved top-level command dispatch and shell completion generation into
  `crates/operon-cli/src/cli_dispatch.rs`.
- Reduced `crates/operon-cli/src/main.rs` to a thin parse-and-dispatch
  entrypoint while preserving the existing command modules and root output
  helper re-exports used by onboard code.
- Added `scripts/verify-v0.17.5-cli-entrypoint-maintainability.sh` and wired it
