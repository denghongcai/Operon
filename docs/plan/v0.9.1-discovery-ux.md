# v0.9.1 Post-v0.9 Discovery UX

Status: Completed.

Goal: improve mDNS discovery ergonomics without reintroducing provider
abstractions.

## Scope

- Make `operon node discover --output-config` safe when the output file already
  exists.
- Add optional health checks for discovered endpoint candidates.
- Document how external control-plane scripts can generate endpoint-only
  config without becoming part of the Operon runtime model.

## Completed

- `operon node discover --output-config <path>` now merges new discovered nodes
  into an existing config and refuses same-node endpoint conflicts instead of
  silently overwriting the file.
- Added `operon node discover --check-health` for best-effort runtime health
  checks on discovered endpoints before printing results.
- Discovery export still writes endpoint-only client nodes and never adds
  provider metadata or policy grants.
- Added [`scripts/verify-post-v0.9-discovery-ux.sh`](../../scripts/verify-post-v0.9-discovery-ux.sh).
- Added README, AGENTS, development phase, and CI validation updates.

## Remaining

- No v0.9.1 discovery UX work remains.
