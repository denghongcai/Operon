# v0.6.9 CLI Contract Cleanup

## Goal

Make the CLI reliable as the supported human, ops, and script interface after
the HTTP facade removal.

## Scope

- `operon --json job run` emits one valid JSON document in non-detached mode.
- non-detached `operon job run` returns non-zero when the remote job reaches a
  failed, cancelled, or timed-out terminal state.
- `operon job logs` honors global `--json` and `--quiet`.
- `operon audit show --json` applies the same filters as text output.
- daemon health reports the public protocol/release line instead of the crate
  package placeholder version.
- `operon init config` writes a starter `config.yaml` plus referenced token and
  secrets files so the generated config can boot directly.
- CI covers the script-facing CLI contracts.

## Non-Goals

- Reintroducing an HTTP runtime API.
- Changing the gRPC runtime schema for version metadata in this cleanup.
- Implementing a new release process.
