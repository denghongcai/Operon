# v0.13 Release Publication and Public Verification

Status: Planned.

## Goal

Publish the current cross-platform preview release from `main` and verify the
public artifacts end to end instead of stopping at source-tree CI.

## Scope

1. Release preflight.
   - Ensure the release commit is merged to `main`.
   - Confirm public release versions, `PROTOCOL_VERSION`, Rust crate versions,
     TypeScript SDK package metadata, README, and release docs are aligned.
   - Confirm CI and CodeQL are green on the release commit.

2. Draft release execution.
   - Create the public release tag from `main`.
   - Let the `Draft Release` workflow build Linux, macOS, Windows, and SDK
     artifacts.
   - Inspect workflow output, release assets, and checksums before publishing.

3. Public artifact verification.
   - Publish the GitHub Release only after draft assets are present.
   - Run `Verify Release Artifacts` against the public tag on Linux, macOS, and
     Windows runners.
   - Run README Quickstart release verification against the public release.

4. Documentation closeout.
   - Record the release tag, validation commands, and any known platform caveats
     in `docs/plan/development-phases.md`.
   - Keep Windows PTY validation deferred unless v0.13.1 changes that status.

## Non-Goals

- No npm or crates.io publication.
- No code signing, notarization, MSI/pkg installers, Homebrew, or package
  manager automation.
- No mount adapter strategy or implementation work.

## Acceptance Criteria

- The release tag is based on `main`.
- `CI` and `CodeQL` are green for the release commit.
- The public GitHub Release contains the expected Linux, macOS, Windows, SDK,
  and checksum assets.
- `Verify Release Artifacts` passes for the public tag.
- README Quickstart release verification passes against the public release.
