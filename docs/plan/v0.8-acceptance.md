# v0.8 Acceptance

v0.8 is scoped to Agent Integration.

It includes:

- an agent-facing tool contract.
- tool schemas for node inspection, capability inspection, fs, jobs, services,
  audit, traces, and execution graphs.
- policy-aware error mapping.
- audit and trace visibility for agent actions.
- at least one runnable agent workflow example.

It deliberately does not include a separate agent runtime, a parallel control
plane, browser UI, or bypasses around Operon policy.

## Validation

The canonical v0.8 validation should run:

```bash
cargo fmt --check
cargo check --workspace --locked
cargo test --workspace --locked
cargo clippy --workspace --locked -- -D warnings
pnpm typecheck
pnpm -r test
scripts/verify-v0.8-agent.sh
```

The agent validation must prove:

- an agent can list configured nodes.
- an agent can inspect capabilities.
- an agent can run a constrained fs/job/service workflow.
- denied agent actions return structured policy errors.
- agent actions are visible in audit output.
- workflow execution produces trace output.

## Known Limits

- v0.8 reuses existing runtime APIs and should not create a separate control
  plane.
- Destructive operations should remain opt-in and policy-gated.
