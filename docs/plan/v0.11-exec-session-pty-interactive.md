# v0.11 Exec Session / PTY Interactive

Status: Completed.

Goal: add a true interactive execution surface for terminal-like workflows
without changing the existing non-interactive `exec.run` behavior.

## Scope

1. Runtime protocol.
   - Add `OpenExecSession` as a bidirectional streaming RPC.
   - The first client message must carry session start metadata.
   - Later client messages may carry raw input bytes or terminal resize events.
   - Server events must report session start, raw terminal output, and terminal
     exit.

2. Policy and capability model.
   - Add `exec.session` authorization semantics distinct from `exec.run`.
   - Advertise the `session` permission only when policy allows interactive
     sessions.
   - Keep `run`, `cancel`, and `logs` permissions unchanged.

3. Daemon runtime.
   - Spawn session commands under a PTY/TTY on supported platforms.
   - Preserve workspace cwd containment, timeout limits, secret authorization,
     audit identity, cancellation, and terminal exec status recording.
   - Record session lifecycle audit events without recording sensitive input
     bytes as audit event resources.

4. CLI and SDK.
   - Add an `operon exec session` command.
   - Stream local stdin to the session and remote PTY output to local stdout.
   - Add SDK helpers for building and consuming exec session streams.

5. Documentation and validation.
   - Update README, `PROTOCOL.md`, runtime API docs, repo-local skills, and
     AGENTS.md when affected.
   - Add `scripts/verify-v0.11-exec-session.sh` and wire it into CI and full
     validation.
   - Document platform support explicitly. The first implementation may be
     Unix-first as long as Windows ConPTY follow-up scope is explicit.

## Non-Goals

- No remote desktop, SSH wrapper, VPN, or network mesh behavior.
- No change to `exec.run`, existing stdin streaming, logs, cancellation, or
  graph execution semantics.
- No automatic capture of interactive input into audit logs.

## Done When

- Active proto exposes `OpenExecSession` and session request/event envelopes.
- `operon config explain` and `operon capability list/explain` can show
  session permission and decisions.
- `operon exec session` can run an interactive command through a PTY on the
  supported first platform set.
- The TypeScript SDK exposes session stream helpers.
- Docs and repo-local skills explain when to use `exec.run` versus
  `exec.session`.
- `scripts/verify-v0.11-exec-session.sh` passes and is wired into CI.

## Completed

- Added `OpenExecSession` bidirectional streaming protocol with start, input,
  resize, started, output, and exit envelopes.
- Added `policy.exec.allow_sessions`, `exec:default` `session` diagnostics,
  and policy-derived `session` capability permissions.
- Added daemon PTY-backed session execution through `portable-pty`, preserving
  cwd containment, timeout, secret authorization, audit, cancellation, exec
  records, and log capture.
- Added `operon exec session` and TypeScript SDK `openExecSession`.
- Bumped Rust crate versions, TypeScript SDK package version, and
  `PROTOCOL_VERSION` to `v0.11.0` / `0.11.0`.
- Updated README, `PROTOCOL.md`, runtime API docs, repo-local skills,
  DEVELOPMENT.md, CI, and validation coverage.
