# v0.17.1 Maintainability Cleanup

Status: Completed.

## Goal

Reduce the next set of high-churn files without changing protocol behavior,
public CLI shape, SDK public API, or mount semantics.

## Scope

1. Split the TypeScript SDK public entrypoint by internal domain while keeping
   `packages/sdk-js/src/index.ts` as the exported API surface.
2. Split Windows mount adapter callback responsibilities in
   `crates/operon-mount/src/windows.rs` behind smaller helper modules.
3. Continue daemon runtime/router cleanup in `crates/operond/src/main.rs` and
   related runtime modules where responsibilities remain mixed.
4. Keep Linux/macOS FUSE adapter behavior stable while isolating high-risk
   `fuse_fs.rs` helpers only when tests can cover the boundary.
5. Add focused validation scripts for the moved boundaries and wire them into
   the narrowest existing validation group.

## Acceptance

- Behavior-preserving moves only; no protocol/schema/version changes in this
  phase.
- Public SDK imports, CLI help, daemon runtime behavior, and mount smoke
  validation remain stable.
- Line-count and responsibility reductions are visible in the touched modules,
  with validation covering the new module boundaries.
