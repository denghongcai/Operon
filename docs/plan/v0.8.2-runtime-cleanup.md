# v0.8.2 Runtime Cleanup

Status: Completed.

This cleanup phase addresses concrete review findings that were small enough to
fix without changing Operon's public runtime model.

Completed:

- unified audit timestamp representation on `u64`, matching the gRPC
  `uint64` schema and removing lossy `u128 as u64` conversion.
- extracted CLI token generation and owner-only private-file writing into a
  shared helper used by both `operon init config` and `operon onboard`.
- made UDP service forwarding abort and await its local socket read task during
  cleanup.
- added explicit service action permissions for `check` and `forward`, and
  enforced them in service health checks plus TCP/UDP forwarding.
- documented empty file write chunks and TCP/UDP tunnel close semantics in
  `PROTOCOL.md`.
- added focused tests for audit timestamp round-tripping and shared private
  file helpers.

Deferred hardening items:

- `init config` and `onboard` have different LAN advertisement defaults because
  they target different workflows. A future config UX phase should make that
  distinction explicit in docs and command output.
- `AppState` remains a broad daemon state container. A later daemon
  maintainability phase should split audit, job runtime, job logs, and service
  state into narrower structs.
