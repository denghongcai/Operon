# v0.6.7 Acceptance Plan

Status: Completed.

v0.6.7 is a runtime infrastructure cleanup milestone before v0.7 TUI work. It
does not add new product capabilities. It hardens existing job lifecycle, job
log streaming, and CLI async execution boundaries.

## Scope

v0.6.7 includes:

- Linux job process-group termination for cancel and timeout.
- Binary-safe job log protocol and SDK/CLI handling.
- Explicit async CLI runtime ownership at the CLI entrypoint.

v0.6.7 does not include:

- TUI console implementation.
- policy package extraction.
- `openat2` workspace traversal.
- protocol-level `argv[]` job execution.
- endpoint discovery beyond local mDNS.

## Acceptance Criteria

Job process lifecycle:

- cancelling a job kills the direct child and shell-spawned descendants on
  Linux.
- timing out a job uses the same process-group termination path.
- terminal job status remains `Cancelled` or `TimedOut` as appropriate.
- stdout/stderr capture still drains before final job completion is recorded.

Binary job logs:

- daemon captures stdout/stderr as bytes, not lossy UTF-8 strings.
- `ListJobLogs` and `StreamJobLogs` expose binary-safe data consistently.
- CLI streaming writes log bytes directly.
- TS SDK `streamJobLogs` yields `ReadableStream<Uint8Array>` without string
  re-encoding.
- JSON/human display paths document or explicitly handle decoding.

CLI async runtime:

- CLI entrypoint owns Tokio runtime explicitly.
- [`operon-cli/src/grpc.rs`](../../crates/operon-cli/src/grpc.rs) no longer contains a singleton runtime or internal
  `block_on`.
- gRPC helper functions are async and awaited by command handlers.
- execution graph request context metadata still reaches daemon audit logs.

## Validation

Required validation commands:

```bash
cargo fmt --check
cargo clippy --workspace --locked -- -D warnings
cargo test --workspace --locked
pnpm --filter @operon/sdk typecheck
pnpm --filter @operon/sdk test
scripts/verify-v0.6.7-runtime.sh
```

Runtime validation should include a Docker-backed or Linux-local job that:

- starts a shell command which spawns a long-lived descendant.
- cancels or times out the job.
- verifies the descendant is no longer running.

Binary log validation should include stdout/stderr bytes that are not valid
UTF-8 and verify they survive daemon capture and client streaming.

## Remaining After v0.6.7

- v0.7 CLI TUI console.
- policy/config package boundary cleanup.
- Linux `openat2` or fd-relative workspace traversal.
- optional protocol-level `argv[]` shell-free job execution.
