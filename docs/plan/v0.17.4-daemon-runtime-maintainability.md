# v0.17.4 Daemon Runtime Maintainability Cleanup

Status: Completed.

## Goal

Reduce daemon exec runtime file responsibility without changing exec protocol,
policy, audit, log, cancellation, or streaming behavior.

## Scope

1. Move exec child process group, Windows Job Object, cancellation, stdio pump,
   and capture-task helpers out of `crates/operond/src/exec_runtime.rs`.
2. Keep `exec_runtime.rs` focused on exec authorization, state transitions,
   log/event APIs, completion, and persistence behavior.
3. Preserve Unix process-group cancellation, Windows Job Object cancellation,
   timeout behavior, stdin pumping, stdout/stderr capture, and existing tests.
4. Add focused validation for the new daemon runtime boundary.

## Acceptance

- `crates/operond/src/exec_process.rs` owns exec child process lifecycle and
  stdio task helpers.
- `exec_runtime.rs` no longer owns `ExecChildGroup`, platform termination
  helpers, or stdio pump/capture helpers.
- Workspace tests and the focused daemon runtime validation pass.
