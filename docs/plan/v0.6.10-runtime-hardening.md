# v0.6.10 Runtime Hardening

Status: Completed.

Goal: close the validated runtime hardening issues found after the v0.6.9 CLI
contract cleanup.

Scope:

- make JSONL store appends safer and durable enough for job/audit metadata.
- constrain daemon store paths to safe config-local files.
- add job completion audit events.
- include concrete spawn errors in job stderr logs.
- validate fs range writes and align `mkdir` behavior.
- keep pagination metadata in the Rust core list models.
- handle mDNS removal events during LAN discovery and stop silently ignoring
  discovery receiver failures.

Done when:

- store writes use secure append semantics and sync written records.
- unsafe store paths are rejected during daemon startup.
- job start and terminal state are both auditable.
- fs range writes reject oversized chunks, overflow, and configured max file
  bounds.
- `mkdir` behavior is explicit and tested.
- core/protocol list conversions preserve `next_page_token`.
- LAN discovery removes services that disappear during the discovery window.
- focused unit tests and workspace validation pass.

Completed:

- [`operon-store`](../../crates/operon-store) now opens JSONL store files with secure append semantics,
  rejects symlink store files on Unix, writes owner-only files, and syncs each
  appended record.
- `operond` rejects daemon store paths that resolve outside the config
  directory or point at symlinks/special files.
- job terminal states now emit audit events with run/step context.
- spawn failures now include the concrete OS error in stderr logs.
- fs range writes reject oversized chunks, offset/data overflow, and writes
  beyond the daemon maximum fs object size; truncate uses the same size bound.
- `MkdirFs` creates missing parent directories consistently with other write
  paths.
- Rust core list models now preserve `next_page_token` through protocol
  conversions.
- LAN mDNS discovery removes services that disappear during the discovery
  window and surfaces receiver failures instead of silently ignoring them.
- CI includes [`scripts/verify-v0.6.10-runtime-hardening.sh`](../../scripts/verify-v0.6.10-runtime-hardening.sh).

Remaining:

- no open v0.6.10 runtime hardening items.
