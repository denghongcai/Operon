# v0.16 Mount Runtime UX Hardening

Status: Completed.

## Goal

Make live mount setup failures easier to understand across Linux, macOS, and
Windows now that live mount is a core cross-platform capability.

## Scope

- Extend `operon doctor` platform output beyond static adapter labels.
- Surface mount runtime status and install hints for Linux FUSE, macOS FUSE-T,
  and Windows WinFsp.
- Add platform-specific mount failure hints to `operon mount` errors.
- Keep filesystem RPCs as the authoritative cross-platform API; runtime checks
  are for OS live mount adapters only.
- Align README, PROTOCOL, AGENTS, and validation coverage with the updated UX.

## Completed

- Added `mount_runtime` and `mount_hint` to `operon doctor` JSON and human
  output.
- Linux doctor checks `/dev/fuse` and `fusermount3`/`fusermount` availability.
- macOS doctor checks common FUSE-T library locations or `pkg-config fuse`.
- Windows doctor checks common WinFsp runtime library installation locations.
- `operon mount` now appends platform-specific setup hints when adapter startup
  fails.
- Updated README and PROTOCOL troubleshooting/runtime language.
- Added validation coverage through
  `scripts/verify-v0.16-mount-runtime-ux-hardening.sh`.

## Remaining

- No v0.16 mount runtime UX hardening work remains.
