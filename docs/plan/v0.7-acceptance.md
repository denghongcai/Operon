# v0.7 Acceptance

v0.7 is scoped to a CLI TUI console.

It includes:

- `operon console`.
- terminal views for nodes, capabilities, jobs, traces, audit, and services.
- filtering and drill-down for audit and trace records.
- job log inspection.
- service health inspection.
- reuse of existing CLI config, auth, endpoint resolution, and runtime clients.

It deliberately does not include a separate graphical UI, separate frontend
server, new runtime API surface, remote desktop, screen streaming, or clipboard
sync.

## Validation

The canonical v0.7 validation should run:

```bash
cargo fmt --check
cargo check --workspace --locked
cargo test --workspace --locked
cargo clippy --workspace --locked -- -D warnings
scripts/verify-v0.7-console.sh
```

The console validation must prove:

- `operon console` starts with an example node config.
- view models render node, capability, job, trace, audit, and service data.
- audit and trace filters behave consistently with non-interactive CLI commands.
- the console can inspect a Docker two-node runtime without requiring a browser.

## Known Limits

- v0.7 is terminal-first and intentionally avoids separate graphical UI work.
- Console smoke tests may validate view models and startup rather than full
  terminal rendering in every CI environment.
