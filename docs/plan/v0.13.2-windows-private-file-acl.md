# v0.13.2 Windows Private File ACL Enforcement

Status: Planned.

## Goal

Move Windows token/config private-file handling from warning-only diagnostics to
real ACL-aware validation where practical.

## Scope

1. ACL model.
   - Define the Windows ACL conditions Operon treats as private enough for
     token/config files.
   - Keep Unix owner-only mode checks unchanged.
   - Decide how to handle inherited permissions, Administrators, SYSTEM, and
     the current user.

2. Implementation.
   - Add Windows ACL inspection for sensitive files generated or consumed by
     CLI/onboard flows.
   - Preserve clear warning/error wording for files that cannot be verified.
   - Keep cross-platform private-file helpers behind a narrow boundary.

3. Operator diagnostics.
   - Update `operon doctor` to report verified Windows ACL status or actionable
     remediation.
   - Keep JSON output stable and script-friendly.

4. Validation and docs.
   - Add unit coverage for Windows ACL classification.
   - Add Windows CI smoke where feasible.
   - Update README, `PROTOCOL.md`, runtime API docs, and plan docs with the
     implemented guarantee.

## Non-Goals

- No general-purpose Windows permission management UI.
- No secret manager integration.
- No mount adapter work.

## Acceptance Criteria

- Windows private-file status is verified where the OS exposes enough ACL
  information.
- Warning-only behavior remains only for cases that cannot be safely verified.
- `operon doctor` reports actionable Windows private-file status.
- Unix private-file behavior is unchanged.
