# v0.16.7 Release Publication and Public Verification

Status: In Progress.

## Goal

Publish the current v0.16 release line from `main` after the v0.18.5 through
v0.18.11 release/install hardening work, then verify public artifacts, install
usability, service-management smoke, and README Quickstart against the
published release.

## Scope

1. Confirm the release commit is merged to `main` before tagging.
2. Align Rust crate versions, TypeScript SDK package version,
   `PROTOCOL_VERSION`, CLI `--version`, release dry-run inputs, and README
   release-maintainer examples to `0.16.7` / `v0.16.7`.
3. Run main CI and CodeQL on the release commit.
4. Run the manual cross-platform live mount release gates and Windows Runner
   Image Smoke on the release commit.
5. Verify pre-tag gates with
   `scripts/release-gate-orchestrate.sh pretag v0.16.7 <commit>`.
6. Push tag `v0.16.7`, let the Draft Release workflow build artifacts, publish
   the release, and verify public artifacts.
7. Run post-publication `Verify Release Artifacts`, `Verify Release Install
   Usability`, and `Verify README Quickstart` workflows against `v0.16.7`.
8. Verify post-release gates with
   `scripts/release-gate-orchestrate.sh postrelease v0.16.7 <commit>`.
9. Record run IDs, tag, commit, release URL, and remaining limits in this phase
   plan, the phase tracker, and AGENTS.md.

## Acceptance

- The release tag is created from the intended `main` commit.
- CI, CodeQL, live mount release gates, Windows Runner Image Smoke, Draft
  Release, release artifact verification, downloaded release install usability,
  downloaded service-management smoke, and README Quickstart verification pass
  for the published release.
- README, PROTOCOL, AGENTS, and phase docs agree on the released version line.
