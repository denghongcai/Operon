# v0.17 Release and CI Observability Cleanup

Status: Completed.

## Goal

Move CI/release environment differences into explicit validation so platform or
tooling drift is caught before a push or release gate blocks late.

## Scope

1. Add a focused validation path for CI validation mode, including
   `OPERON_SKIP_SDK_TESTS=1`, so SDK validation scripts do not require pnpm in
   jobs where TypeScript CI already owns SDK tests.
2. Add a cross-target test compilation check for Windows-only daemon tests so
   `#[cfg(windows)]` imports and helper boundaries are checked without waiting
   for the Windows platform smoke job.
3. Add release workflow observability notes for archive smoke, live mount
   release gates, public artifact verification, and README Quickstart
   verification.
4. Keep validation scripts grouped through `scripts/ci/run-validations.sh`
   instead of adding version-specific workflow jobs.
5. Document failure triage expectations: pull logs as soon as the failing job
   identifies the issue, cancel obsolete runs, and rerun only after a targeted
   fix.

## Acceptance

- The new validation catches CI-mode SDK checks and Windows-only test
  compilation locally or in the narrowest existing CI group.
- Release/CI docs explain where to find the relevant logs and when to cancel a
  run after a deterministic failure.
- Main CI remains grouped by capability/OS instead of by version number.
