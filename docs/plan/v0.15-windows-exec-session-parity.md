# v0.15 Windows Exec Session Parity

Status: Completed.

## Goal

Promote Windows interactive exec sessions from an explicit unsupported decision
to a supported `portable-pty` path while keeping the existing
`OpenExecSession` protocol shape unchanged.

## Scope

1. Daemon session support.
   - Remove the Windows-only `UNIMPLEMENTED` guard before opening
     `exec session`.
   - Keep `portable-pty` as the only PTY abstraction.
   - Keep command-string shell defaults platform-specific: `cmd.exe /C` on
     Windows and `/bin/sh -c` on Unix-like platforms.

2. Runner-safe validation.
   - Run the daemon PTY smoke on Windows as well as Unix-like platforms.
   - Keep the PTY smoke bounded so Windows CI cannot hang indefinitely if the
     backend regresses.

3. Operator UX and docs.
   - Update `operon doctor` to report Windows PTY support instead of
     `windows-exec-session-unsupported`.
   - Update README, `PROTOCOL.md`, runtime API docs, architecture notes, and
     older validation scripts whose previous assertions encoded the v0.13.1
     unsupported decision.

4. Version alignment.
   - Align Rust crate versions, the TypeScript SDK package version, CLI version
     output, and `PROTOCOL_VERSION` to `0.15.0` / `v0.15.0`.

## Non-Goals

- No replacement PTY crate beyond `portable-pty`.
- No terminal multiplexer, shell profile management, or remote desktop scope.
- No change to non-interactive exec cancellation; Windows continues to use Job
  Object process-tree cancellation for `exec run`.

## Acceptance Criteria

- Windows no longer returns `UNIMPLEMENTED` before opening an interactive exec
  session.
- The platform smoke workflow runs the bounded portable-pty smoke on Windows.
- `operon doctor` reports Windows PTY session support.
- Current docs no longer describe Windows interactive exec sessions as
  unsupported.
- `scripts/verify-v0.15-windows-exec-session-parity.sh` passes and is wired
  into `scripts/ci/run-validations.sh`.

## Completed

- Removed the Windows-only `Status::unimplemented` guard from daemon exec
  session startup.
- Replaced platform-specific support tests with one
  `exec_session_platform_is_supported` test.
- Made the `exec_session_portable_pty_smoke_outputs_and_exits` test run on
  Windows and added bounded output and child-wait timeouts.
- Released `portable-pty` slave handles after process spawn and made the smoke
  test take and close the PTY writer before waiting for output, matching the
  `OpenExecSession` EOF path after a client finishes sending input.
- Updated CI platform smoke to run the support decision and portable-pty smoke
  on Windows.
- Updated doctor diagnostics and current documentation to report Windows
  `portable-pty` session support.
- Added consolidated validation coverage through
  `scripts/verify-v0.15-windows-exec-session-parity.sh`.
- Aligned the repository version line to `0.15.0` / `v0.15.0`.

## Remaining

- No v0.15 Windows exec session parity work remains.
