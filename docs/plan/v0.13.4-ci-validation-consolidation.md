# v0.13.4 CI Validation Consolidation

Status: Completed.

## Goal

Reduce GitHub Actions job sprawl by consolidating version-scoped validation jobs
into a small grouped validation matrix while keeping each validation script
separate and easy to maintain.

## Scope

1. Validation runner.
   - Add a single runner script that executes existing `scripts/verify-*.sh`
     validation scripts in a stable order.
   - Preserve script files as the maintenance boundary for phase-specific
     checks.
   - Use GitHub Actions log groups for each validation script.
   - Collect all failing validations and report a summary at the end instead of
     stopping at the first failure.

2. Workflow simplification.
   - Keep `Rust`, `TypeScript`, and platform smoke jobs separate because they
     validate different runtimes and operating systems.
   - Replace the version-scoped validation matrix with grouped `Validation`
     jobs for `core`, `runtime`, `sdk`, and `linux-system`.
   - Depend on the `TypeScript` job and set `OPERON_SKIP_SDK_TESTS=1` in CI so
     validation scripts do not rerun `@operon/sdk` tests already covered by
     `pnpm -r test`.

3. Future rules.
   - New version validation scripts should be added to the consolidated runner,
     assigned to the narrowest existing group, and not added as new
     version-specific GitHub Actions matrix entries.
   - SDK validation scripts may keep local `@operon/sdk` checks, but CI should
     skip duplicates when the `TypeScript` job already runs the same tests.
   - Add a separate CI job only when the validation needs a materially
     different OS, permissions model, service container, or trigger.

## Non-Goals

- No removal of individual validation scripts.
- No merge of Rust, TypeScript, and platform smoke jobs into the validation job.
- No change to release-draft or release-artifact verification workflows.

## Completed

- Added `scripts/ci/run-validations.sh` as the consolidated validation runner.
- Replaced the `.github/workflows/ci.yml` version validation matrix with four
  grouped `Validation` jobs.
- Set validation CI to skip duplicate `@operon/sdk` tests already covered by
  the `TypeScript` job.
- Updated contributor and agent guidance so future version validation additions
  extend the runner and choose an existing group instead of adding more
  version-specific matrix jobs.

## Acceptance Criteria

- `CI` exposes grouped `Validation` jobs instead of one job per version
  validation script.
- Individual validation scripts remain independently runnable.
- A failed validation script appears as its own log group and in the final
  failure summary.
- Validation CI skips duplicate SDK unit tests after the `TypeScript` job has
  run `pnpm -r test`, while local validation scripts still run them by default.
- Future CI rules are documented in `DEVELOPMENT.md`, `AGENTS.md`, and
  `docs/plan/development-phases.md`.
