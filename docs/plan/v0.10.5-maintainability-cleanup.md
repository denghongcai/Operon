# v0.10.5 Maintainability Cleanup

Status: Completed.

Goal: continue behavior-preserving module cleanup after the exec session split
by moving service tunnel state machines and CLI service transport helpers behind
focused module boundaries.

## Scope

1. Daemon TCP service tunnel boundary.
   - Move TCP service tunnel stream handling out of
     `crates/operond/src/service_forward.rs`.
   - Keep service authorization and health-check policy decisions in
     `service_forward.rs`.

2. Daemon UDP/datagram service tunnel boundary.
   - Move datagram tunnel session state, UDP socket connection, idle pruning,
     and close response helpers out of `service_forward.rs`.
   - Preserve the existing `OpenServiceDatagramTunnel` protocol behavior.

3. CLI service transport helper boundary.
   - Move CLI service forwarding gRPC helpers out of the broad
     `crates/operon-cli/src/grpc.rs` surface.
   - Keep public CLI command behavior and output unchanged.

4. Validation.
   - Add `scripts/verify-v0.10.5-maintainability-cleanup.sh`.
   - Wire the validation script into CI and DEVELOPMENT.md.

## Non-Goals

- No service protocol changes.
- No service policy vocabulary changes.
- No onboarding refactor in this phase.

## Done When

- TCP tunnel stream behavior lives in a focused daemon module.
- UDP/datagram tunnel session behavior lives in a focused daemon module.
- CLI service forwarding transport helpers live in a focused module.
- Existing service forwarding validation still passes.
- `scripts/verify-v0.10.5-maintainability-cleanup.sh` passes and is wired into
  CI.

## Completed

- Added `crates/operond/src/service_tcp_forward.rs` for daemon TCP service
  tunnel stream behavior.
- Added `crates/operond/src/service_datagram_forward.rs` for daemon UDP
  datagram tunnel sessions, UDP socket handling, idle pruning, and close
  response helpers.
- Kept `crates/operond/src/service_forward.rs` focused on service policy,
  health checking, target metadata validation, authorization, and delegation.
- Added `crates/operon-cli/src/grpc_service.rs` for CLI service forwarding
  transport helpers, leaving `crates/operon-cli/src/grpc.rs` focused on shared
  non-service runtime helpers.
- Added `scripts/verify-v0.10.5-maintainability-cleanup.sh` and wired it into
  CI and DEVELOPMENT.md.
