# v0.11.2 Exec Session Hardening

Status: Completed.

Goal: tighten the PTY-backed exec session surface added in v0.11 so
interactive clients preserve terminal dimensions, disconnected response streams
do not leave orphaned remote processes, and Windows ConPTY follow-up work has a
concrete dependency direction.

## Scope

1. CLI terminal sizing.
   - Use the local terminal size as the default `ExecSessionStart` rows/cols
     when `operon exec session` is attached to a TTY.
   - Preserve explicit `--rows` and `--cols` overrides for scripts and tests.

2. CLI terminal resize forwarding.
   - On Unix, listen for terminal resize signals while an interactive session
     is active.
   - Send `ExecSessionResize` messages over the same bidirectional stream.
   - Keep non-TTY, `--content`, and non-Unix paths deterministic and quiet.

3. Session lifecycle hardening.
   - Terminate the remote session when the response stream is dropped before a
     terminal exit event.
   - Preserve normal stdin close behavior when the client intentionally
     half-closes the request stream.

4. Windows ConPTY research.
   - Record whether `zhiburt/conpty` is a plausible future Windows backend.
   - Do not add a Windows runtime backend in this phase unless the dependency
     and CI target are ready.

5. Validation.
   - Add `scripts/verify-v0.11.2-exec-session-hardening.sh`.
   - Wire the validation script into CI and DEVELOPMENT.md.

## Non-Goals

- No Windows ConPTY implementation in this phase.
- No change to the `OpenExecSession` protobuf shape.
- No SSH, VPN, or remote desktop semantics.
- No automatic audit capture of raw interactive input bytes.

## Done When

- `operon exec session` uses local TTY dimensions by default.
- Interactive Unix sessions forward resize events.
- Dropped response streams cancel the remote session before terminal exit.
- Docs record the `zhiburt/conpty` assessment and future Windows scope.
- `scripts/verify-v0.11.2-exec-session-hardening.sh` passes and is wired into
  CI.

## Completed

- `operon exec session` now defaults to the attached local TTY dimensions when
  `--rows` or `--cols` are omitted, while preserving explicit overrides for
  scripts and tests.
- Interactive Unix sessions now listen for terminal resize signals and forward
  them as `ExecSessionResize` messages over the existing
  `OpenExecSession` bidirectional stream.
- The daemon session response stream now owns a `SessionStreamGuard`; if the
  response stream is dropped before a terminal exit event, the guard sends
  `SessionControl::Terminate` so the remote PTY process is not orphaned.
- Reviewed `https://github.com/zhiburt/conpty` and crates.io package
  `conpty = "0.7.0"` as the likely future Windows backend. It provides a
  Windows ConPTY interface and is plausible for a later Windows adapter, but it
  should be introduced behind a platform-specific backend boundary after
  Windows CI and packaging are defined.
- Added `scripts/verify-v0.11.2-exec-session-hardening.sh` and wired it into
  CI and DEVELOPMENT.md.

## Remaining

- No v0.11.2 work remains.
- Windows ConPTY implementation remains future platform/distribution work.
