# v0.8.16 Endpoint Model Simplification

Status: Completed.

Goal: remove the provider abstraction from Operon's user-facing endpoint model.

Decision:

- Operon consumes explicit gRPC endpoints. It does not need to know whether an
  endpoint is reachable through Cloudflare Mesh, Tailscale, WireGuard, SSH,
  Kubernetes DNS, LAN, or another private network.
- mDNS remains a convenience mechanism for discovering candidate LAN endpoints,
  not a provider type.
- External network systems solve reachability. Operon starts at
  `node_id -> endpoint` and owns capability policy, execution, audit, and
  traces.

Completed:

- Removed `NetworkProviderKind` and `provider` from `NodeEndpoint` and
  `NodeConfig`.
- Left stale `provider` config fields inert: they are not emitted, surfaced, or
  consumed when resolving endpoints.
- Removed provider from mDNS discovery records and mDNS advertisement metadata.
- Removed the `operon provider` command and the `--provider` flag from
  `operon node discover`.
- Updated init/onboard generated config and scripts to emit endpoint-only node
  entries.
- Updated the TypeScript SDK `NodeEndpoint` type to endpoint-only.
- Reframed v0.9 planning around endpoint model acceptance and mDNS discovery
  UX instead of non-LAN provider adapters.
- Added `scripts/verify-v0.8.16-endpoint-model-simplification.sh`.

Validation:

```bash
scripts/verify-v0.8.16-endpoint-model-simplification.sh

```

Remaining:

- No v0.8.16 work remains.
- Future discovery work should improve endpoint import/export and mDNS UX, not
  add provider-specific runtime behavior.
