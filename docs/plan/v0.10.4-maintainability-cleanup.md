# v0.10.4 Maintainability Cleanup

Status: Completed.

Goal: keep the runtime and CLI surfaces modular after the exec session work by
moving large routing and client-helper responsibilities behind focused module
boundaries without changing behavior.

## Scope

1. Daemon runtime routing.
   - Move exec RPC request handling out of `crates/operond/src/main.rs`.
   - Keep `main.rs` focused on daemon startup, service registration, auth, and
     high-level delegation.

2. Exec session runtime ownership.
   - Keep PTY/session state-machine code separate from non-interactive
     `exec.run` process execution code.
   - Keep session stream envelope handling testable without reading
     `main.rs`.

3. CLI transport helpers.
   - Split exec-specific gRPC client helpers out of the broad
     `crates/operon-cli/src/grpc.rs` surface.
   - Preserve existing command behavior and output contracts.

4. Onboarding and service forwarding.
   - Do not refactor unrelated behavior unless v0.11 work touches it.
   - Leave future onboarding/service decomposition explicit if it remains
     outside this cleanup.

5. Validation.
   - Add `scripts/verify-v0.10.4-maintainability-cleanup.sh`.
   - Wire the validation script into CI and full validation.

## Non-Goals

- No protocol behavior changes in this cleanup phase.
- No broad crate split for policy, discovery, service forwarding, or
  onboarding unless needed by the exec session implementation.
- No compatibility aliases for removed `job` terminology.

## Done When

- `operond/src/main.rs` delegates exec RPC behavior to a focused module.
- PTY/session runtime code lives in a dedicated daemon module.
- CLI exec gRPC helpers live behind a focused module boundary.
- Existing validation still passes.
- `scripts/verify-v0.10.4-maintainability-cleanup.sh` passes and is wired into
  CI.

## Completed

- Added `crates/operond/src/exec_service.rs` so `operond/src/main.rs`
  delegates exec RPC behavior instead of owning request parsing, stream
  ordering, log streaming, stdin handling, cancellation, and session routing.
- Added `crates/operond/src/exec_session.rs` for PTY/session lifecycle and
  state-machine behavior.
- Added `crates/operon-cli/src/grpc_exec.rs` for exec-specific streaming CLI
  transport helpers.
- Added `scripts/verify-v0.10.4-maintainability-cleanup.sh` and wired it into
  CI and DEVELOPMENT.md.

## Remaining

- No v0.10.4 work remains.
