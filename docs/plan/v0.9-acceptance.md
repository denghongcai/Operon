# v0.9 Acceptance

v0.9 is scoped to non-LAN provider discovery adapters.

It includes:

- a common provider discovery result model.
- discovery behavior for at least one non-LAN provider.
- mocked provider API tests.
- clear merge behavior between discovered endpoints and manual node config.
- documentation for credentials, caching, and provider limits.

It deliberately does not include NAT traversal, relay network infrastructure,
VPN/device mesh IP assignment, subnet routing, global routing, or automatic
capability authorization for discovered nodes.

## Validation

The canonical v0.9 validation should run:

```bash
cargo fmt --check
cargo check --workspace --locked
cargo test --workspace --locked
cargo clippy --workspace --locked -- -D warnings
scripts/verify-v0.9-provider-discovery.sh
```

The provider discovery validation must prove:

- mocked provider responses can produce node endpoint records.
- discovered endpoints can be listed before use.
- manual endpoint config remains supported and can override discovery.
- discovery failures do not break manual nodes.
- discovered nodes do not automatically receive capability authorization.

## Known Limits

- Live Cloudflare, Tailscale, or Kubernetes validation may remain optional and
  documented separately from CI.
- Provider adapters discover endpoints only; connectivity remains external to
  Operon.
